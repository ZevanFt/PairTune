# PairTune UI 现状说明与下一步建议

更新时间：2026-03-09

## 1. 当前已完成（可交付）
1. 四个主页面 AppBar 已统一：`[状态点] 标题`，仅图标无文字。
2. 全站核心页面已收敛到统一浅色语义色变量（`AppTheme`）：
- 首页
- 商城
- 通知
- 我的
- 编辑任务
- 调试页
- 启动页与底部导航
3. 重输入弹窗已页面化：
- 发布商品 / 编辑商品 -> 独立页面
- 编辑昵称 -> 独立页面
4. 确认动作已统一为底部面板风格：
- 兑换确认
- 下架确认
5. 首页筛选排序已从拥挤条改为“简洁工具条 + 底部筛选面板”。

## 2. 当前视觉基线（已经稳定）
1. 视觉方向：简约浅色、低饱和、卡片化。
2. 深色区域仅用于 Hero 头卡，不再全页重色。
3. 卡片体系统一：`panel + panelBorder`。
4. 错误/警告提示统一：`error/warn` 语义色。
5. 列表加载策略统一：只在数据列表使用骨架屏，页面结构常驻。

## 3. 仍可优化（仅 3 项，按优先级）
1. 全局动效收敛（优先）
- 为底栏切换、卡片入场、面板弹出统一时长与缓动曲线。
- 目标：减少“静态拼接感”。

2. 字体层级微调
- 统一标题/副标题/正文/标签字重与字距，减少页面间密度差异。
- 目标：增强品牌一致性与阅读节奏。

3. 空状态插画与语义文案
- 首页空任务、商城空商品、通知空列表统一语义与插画风格。
- 目标：提升“无数据时”体验完整度。

## 4. 进入开发后端前的 UI 冻结建议
建议将当前 UI 作为 `v1-beta` 视觉基线冻结，后续仅做：
1. bug fix
2. 颜色变量微调
3. 动效微调
避免在后端联调阶段继续大改布局。
