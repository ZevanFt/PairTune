# PairTune UI 重设计规范（v1）

更新时间：2026-03-09
目标：简约、浅色、克制；双人协作为卖点，单人可独立成立。

## 1. 设计原则
1. 浅色主导：大面积留白 + 低饱和色块，深色只用于 Hero 或关键强调。
2. 信息分层：页面最多三层视觉权重（标题层、卡片层、辅助层）。
3. 组件统一：卡片、按钮、输入框、标签、弹窗只允许一套视觉语言。
4. 稳定结构：网络失败时保留页面结构，不整屏骨架。
5. 可商用依赖：仅使用开源可商用组件与图标。

## 2. 页面清单（冻结）
1. 欢迎/模式选择（单人 or 双人）
2. 任务首页（四象限 + 今日聚焦 + 列表）
3. 积分商城（积分、可兑换、我发布、记录）
4. 通知中心
5. 我的
6. 新建/编辑任务
7. 调试页

## 3. 统一视觉规则
1. 颜色
- `primary`: 品牌主蓝（用于选中态、关键按钮）
- `bg`: 三段浅色背景渐变（极低对比）
- `panel`: 卡片统一暖白
- `border`: 统一浅边框
- `state`: success/warn/danger 仅用于状态提示

2. 圆角与阴影
- 页面主卡：16-22
- 小标签/按钮：10-14
- 阴影：仅 1 层，透明度低，不做重阴影

3. 字体层级
- 页面标题：20/800
- 分区标题：18/800
- 卡片标题：15-16/700
- 辅助文案：12-13/500-600

4. 间距节奏
- 页面外边距：16
- 分区间距：14-16
- 卡片内间距：12
- 元素间距：6/8/10/12 四档

5. 导航栏
- 底栏保持悬浮胶囊
- 选中态：轻背景 + 顶部短线 + 图标/文字同色

6. AppBar 后端状态
- 四主页面统一为 `[状态点] 标题`
- 仅图标，无文字
- 灰=检测中，绿=在线，红=离线/异常

## 4. 页面重做顺序
1. 任务首页（最核心，先定风格）
2. 商城页（复用首页的卡片语义）
3. 通知页与我的（统一信息密度）
4. 编辑任务与调试页（工具页但保持同体系）
5. 全局弹窗与空状态复核

## 5. 验收标准
1. 任意页面截图中，颜色来源可追溯到主题变量。
2. 页面不存在“原生默认控件拼盘感”。
3. 异常/离线状态不破坏布局。
4. `flutter analyze` 必须通过。
