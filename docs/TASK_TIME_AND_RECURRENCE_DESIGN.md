# 任务时间与重复策略设计（V1）

更新时间：2026-03-08

## 1. 设计目标
- 支持“单次任务 + 重复任务”统一建模。
- 明确“日期截止”和“精确时间截止”的业务语义。
- 让用户能配置常见重复模式：每天、每周、按周几、每月、每年。

## 2. 截止时间语义

### 2.1 `due_mode = day`
- 含义：当天内任意时间完成都算按时。
- 例如：用户选择 `2026-03-09`，实际截止点按本地日历日结束处理，即 `2026-03-09 23:59:59.999`。
- 用途：日常习惯类任务（背单词、运动、读书）。

### 2.2 `due_mode = time`
- 含义：精确到时分的严格截止点。
- 例如：`2026-03-09 10:00`，超过即算逾期。
- 用途：会议、出行、预约等时间敏感任务。

## 3. 重复规则语义

### 3.1 `repeat_type`
- `none`：不重复
- `daily`：每日重复
- `weekly`：按周重复（每 N 周）
- `weekly_custom`：按周几重复（如每周一/三/五）
- `monthly`：按月重复（每 N 月）
- `yearly`：按年重复（纪念日、生日）

### 3.2 `repeat_interval`
- 对 `daily|weekly|monthly|yearly` 生效，最小值 1。
- 例如：`weekly + interval=2` 表示每两周重复一次。

### 3.3 `repeat_weekdays`
- 仅对 `weekly_custom` 生效，范围 `1..7` 对应 `周一..周日`。
- 序列化为字符串（如 `1,3,5`）。

### 3.4 `repeat_until`
- 可选；用于限制自动生成的最后日期上限。
- 超过该日期后不再生成下一次任务。

## 4. 自动生成下一次任务（服务端）
- 触发时机：任务状态从“未完成”变更为“已完成”。
- 生成策略：
  - `daily|weekly|monthly|yearly`：按 `repeat_interval` 递增。
  - `weekly_custom`：计算当前截止点之后最近的目标周几。
- 继承字段：标题、备注、四象限、积分、重复规则。
- 新任务状态：`is_done = 0`。

## 5. 交互建议
- 编辑页提供“截止模式”开关：
  - 当天截止（默认）
  - 精确时间
- 选择 `weekly_custom` 时展示周几多选（Chip）。
- 对“每年重复”推荐用于纪念日，并显示友好文案提示。

## 6. 后续可扩展
- 工作日重复（Mon-Fri 快捷选项）。
- 节假日跳过策略。
- 重复实例链路（母任务 + 子实例）建模，支持更强统计能力。
